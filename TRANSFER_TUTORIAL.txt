TRANSFERRING YOUR PROJECT TO YOUR LOCAL MACHINE
=================================================

This is a comprehensive guide to transferring your project from this environment to your local machine. Since your project is already built with React, Vite, and Tailwind CSS v4, this tutorial will show you how to recreate that exact setup.

Phase 1: Preparation
--------------------
Before you start, ensure you have Node.js (v18 or higher) installed on your computer.
- Create a New Folder: Create a folder on your computer named "gabrielle-custodio-styling".
- Open Terminal: Open your terminal (VS Code Terminal, CMD, or PowerShell) inside that folder.

Phase 2: Project Initialization
-------------------------------
Run the following command to create a fresh Vite project:
npm create vite@latest . -- --template react

When asked to "Proceed", type "y". This will set up the basic React structure.

Phase 3: Installing Dependencies
--------------------------------
You need to install the specific libraries we used. Run this command:
npm install lucide-react motion react-router-dom date-fns clsx tailwind-merge @google/genai cloudinary express better-sqlite3 multer cors dotenv @fullcalendar/react @fullcalendar/daygrid @fullcalendar/interaction @fullcalendar/timegrid

Then, install the Tailwind CSS v4 and TypeScript developer tools:
npm install -D tailwindcss @tailwindcss/vite autoprefixer typescript @types/node @types/express tsx

Phase 4: Configuration Files
----------------------------
This is the most important part to ensure the "Hybrid" (JS + TS) setup works. Copy the content from your files here into your local files:
- vite.config.ts: Copy the content from this file to your local one. It handles the Tailwind v4 integration.
- tsconfig.json: Copy this content to enable the TypeScript "safety net" for your .jsx files.
- package.json: Make sure your scripts section looks like this:

"scripts": {
  "dev": "vite",
  "build": "vite build",
  "lint": "tsc --noEmit",
  "preview": "vite preview"
}

Phase 5: Moving the Source Code
-------------------------------
Now, copy the folders and files from the sidebar here to your local src folder:
- Assets: Create src/assets/images and move all your .jpeg and .jpg files there.
- Components: Copy the entire /src/components/ folder.
- Pages: Copy the entire /src/pages/ folder.
- Constants: Copy src/constants.js.
- Global Styles: Copy src/index.css. Ensure it starts with: @import "tailwindcss";
- The App Entry: Copy src/App.jsx and src/main.jsx.

Phase 6: Environment Variables
------------------------------
Create a file named .env in your root directory (next to package.json). This protects your keys:

GEMINI_API_KEY=your_actual_key_here
VITE_CLOUDINARY_CLOUD_NAME=your_cloud_name
VITE_CLOUDINARY_UPLOAD_PRESET=your_preset

Phase 7: Running the App
------------------------
In your terminal, run:
npm run dev

Vite will provide a link (usually http://localhost:5173). Open that in your browser, and your project will be running locally!

Pro-Tip: The "Hybrid" Advantage
-------------------------------
This setup allows you to use JavaScript (.jsx) for your UI logic while benefiting from TypeScript's (.ts) configuration for modern styling and build tools.
